Locator Design

Responsive web experience revamped

Finding an offline store made easier

According to the feedbacks collected from Reebok customer service, our customers constantly call to require about the offline stores. It took quite some time and patience to accurately deliver the store information over the phone. The current website contains a static page with limited text information on it.

Do you have a store nearby?

This page is totally static without any interactive element, yet the store photo with a store name over it might hint that it is clickable. This is in conflict with user cognition.

As Reebok is opening more and more new stores, the store list is becoming longer than a few scroll on a mobile device. Therefore, users might have to scroll from top to the bottom to find a desired store. This is quite time-consuming…

Introducing the locator

The brand Reebok is expanding in a fast pace in the China market. The store list page needs to be refreshed to meet the new business requirements and user needs. I led the design of the locator both across desktop and mobile device.


Before I dived into design, I did a brief competitive product analysis to help me find the best practice in the design. I also investigated the users to discover their pain points. Meanwhile, I worked closely with PM and engineers to know about the technical constraints and development timeline in this iteration.

Design Principles:


Keep the look and feel consistent across different OS and platforms.


Do not obstruct the view. Create an immersive experience while the users are trying to spot the location in the map.


Do not make the users wait. If the waiting time is over 0.5s, set up a loading animation between the transition.


Strong signifier helps users navigate smoothly through the page.

Intuitive visual design.


Attention to details. Complete interaction design details, e.g. mouse status: mouse over, mouse down, selected, etc.


Consider edge cases.

The full flow of a locator

Edge case - no result

What are the best practices for mobile design? The real estate is limited on a small screen size. Therefore, the hierarchy, layout, size and interaction are all reconsidered to improve the mobile experience. There are two modes to check the store info - list mode and map mode. But how do our users know about it without thinking hard? A toggle switch changes the views back and forth between list mode and map mode. An anchored link on the text leads the users all through the pathway from list to map mode. A “Load More” CTA limits the default height of the list to keep the map mode in view, while gives our users the flexibility to view more in the list… Proper signifiers and intuitive design make the users think less and simplify the flow.

A side note - attention to details

Based on the browser attributes, the scrollbar differs in style on different OS. If you’re still using the way outdated Windows system in your machine, the “classic” scrollbar is apparently not aligned with the brand visual identity. This is why I designed the customized scrollbar to keep the look and feel consistent across different systems. In the new design, the scrollbar is always shown, rather than scroll to show, to strongly signify the overflow part of the list, hinting the users to scroll.
